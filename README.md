# Predicting Personal Film Preferences from Letterboxd Ratings

**MATH 6243: Statistical Learning | Northeastern University | Spring 2026**  

## Overview

This project builds a personal film preference model using Letterboxd ratings data. Given a film's observable attributes (genre, director, release year, critical scores, language), the model predicts whether you will like it (≥ 3.5★). The trained model is then applied to a personal watchlist to produce a ranked priority list of films to watch next.

## Getting Your Letterboxd Data

You will need to export two files from your Letterboxd account:

1. Go to https://letterboxd.com/settings/data/
2. Click **Export your data** — this downloads a zip file
3. Unzip it and locate:
   - `ratings.csv` — your personal film ratings (needed for training)
   - `watchlist.csv` — your saved films to watch (needed for scoring)
4. Place both files in the root of this repository before running the pipeline

> **Note:** The `ratings.csv` included in this repository is my own personal data and is provided as a reference example. Delete it and replace with your own export before running the pipeline. `enriched_ratings.csv` will be generated automatically when you run `enrich.py`.

Note: you need at least a few dozen rated films for the model to learn meaningful patterns. The more ratings you have, the better.

## Running the Pipeline

### Step 1 — Enrich ratings data

You will need a free TMDB API key (https://www.themoviedb.org/settings/api) and a free OMDB API key (https://www.omdbapi.com/apikey.aspx).

```bash
python enrich.py --tmdb YOUR_TMDB_KEY --omdb YOUR_OMDB_KEY
```

This reads `ratings.csv`, queries TMDB and OMDB for each film, and outputs `enriched_ratings.csv`. API responses are cached in `cache/` so re-runs are instant.

To enrich your watchlist:

```bash
python enrich.py --tmdb YOUR_TMDB_KEY --omdb YOUR_OMDB_KEY --watchlist
```

### Step 2 — Exploratory data analysis

Open and run `eda.ipynb` top to bottom. Requires `enriched_ratings.csv` in the working directory.

### Step 3 — Modeling and watchlist scoring

Open and run `modeling.ipynb` top to bottom. Runs feature engineering, nested 5-fold CV, elastic net and random forest models, from-scratch logistic regression, and watchlist scoring. Saves all figures to the working directory.
